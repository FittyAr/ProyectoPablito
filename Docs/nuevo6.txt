# 🧱 1. Arquitectura propuesta

Dado que querés Avalonia + EF Core + SQLite + multiplataforma, lo correcto es:

### ✔️ Clean Architecture (ligera, no overkill)

Separar en proyectos:

```
/src
 ├── App.Desktop (Avalonia UI)
 ├── App.Mobile (futuro)
 ├── App.Web (futuro WASM o SSR)
 │
 ├── Core (dominio puro)
 ├── Application (casos de uso)
 ├── Infrastructure (EF Core + SQLite)
 ├── Shared (DTOs, helpers)
```

### Stack recomendado (actualizado)

* .NET 10
* Avalonia 11+
* EF Core 9+
* SQLite
* CommunityToolkit.Mvvm (para MVVM limpio)
* FluentValidation (sí, muy buena decisión)
* Serilog (logging)
* AutoMapper (opcional, pero útil)
* MediatR (opcional, depende qué tan complejo lo quieras)

# 🧩 3. Modelo de dominio inicial (propuesta)

Esto es clave hacerlo bien ahora.

## Entidad principal: Movimiento

```csharp
public class Movimiento
{
    public Guid Id { get; set; }
    public DateTime Fecha { get; set; }

    public string Descripcion { get; set; } = null!;

    public decimal MontoUnitario { get; set; }
    public int Cantidad { get; set; }

    public decimal Total => MontoUnitario * Cantidad;

    public TipoMovimiento Tipo { get; set; }

    public Guid? CategoriaId { get; set; }
    public Categoria? Categoria { get; set; }
}
```

## TipoMovimiento

```csharp
public enum TipoMovimiento
{
    Gasto,
    Ingreso
}
```

## Categoría

```csharp
public class Categoria
{
    public Guid Id { get; set; }
    public string Nombre { get; set; } = null!;
}
```
